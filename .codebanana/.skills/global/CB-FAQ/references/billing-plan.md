We offer flexible plans designed to scale with your application. Whether you are building a proof of concept or deploying for production, explore the feature breakdown below to find the tier that best aligns with your current needs.

      No feature gating. Access everything, including collaboration, on the free plan.

### Personal

    Supercharge your workflow with dedicated power and privacy for serious individual projects.

    - **300 agent requests** / month

      - Higher request quotas
      - Massive cloud storage
      - Flexible VM tiers
      - Complete environment isolation.

### Professional

    Empower your team with maximum performance and advanced collaboration tools for scaling applications.

    - **1000 agent requests** / month
    - **5GB** cloud storage

      Massive request limits and cloud storage tailored for team collaboration.

  View the price to find the most suitable plan.

## Purchasing a plan

Once you have selected the plan that fits your needs, you can upgrade using the following methods：

  **Have a promo code?** Please apply it **before** proceeding to checkout, as it will not take effect automatically.

**To purchase a plan:**

1. Click the **upgrade button** at the top right of your workspace
2. Choose a suitable Plan
3. Select a preferred billing cycle:
   - **Yearly:** Save with an annual discount.
   - **Monthly:** Pay month to month.
4. Click the **VM tiers** to customize
5. Click **Upgrade** inside the Plan you choose to checkout

## Requests & Token

#### **What is a Request**

A **request** represents a single interaction with the agent.

- Each prompt sent to the agent counts as one request
- Includes:
  - Code generation
  - Q&A
  - Task execution

Plans typically include a fixed number of requests per month, depending on your subscription tier.

#### **What are Tokens**

**Tokens** measure the actual amount of computation used by the model.

- Includes:
  - Input (your prompt, context, files)
  - Output (agent responses, generated code)
- More complex tasks or larger context will consume more tokens

Tokens are used to calculate **usage-based costs**, especially beyond included quotas.

#### **How Billing Works**

Usage is calculated based on both requests and tokens:

1. **Within plan limits** Requests are deducted from your monthly quota
2. **Beyond plan limits** Usage may switch to **pay-as-you-go (token-based billing)**
3. **Team Agent vs Private Agent (My agent before)**
   - Team Agent usage is charged to the **project owner**
   - Private Agent usage is also charged to the **project owner**

#### **Additional Considerations**

- Using more advanced models or larger contexts may increase token consumption
- Some features (e.g. support sessions or enhanced capabilities) may apply **multipliers to usage**
- Usage details can be tracked in the **Usage dashboard**
