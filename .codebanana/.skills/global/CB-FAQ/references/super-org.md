# 超级组织与 CodeBanana 产品理念
> Source: /documentation/about/super-org

出门问问 CEO 超级组织分享会内容摘录，介绍 CodeBanana 的产品背景、核心理念与设计哲学。

---

## 为什么做 CodeBanana？

公司里有两类工作，却一直用两套割裂的工具：

| 工具类型 | 代表产品 | 用途 |
|----------|----------|------|
| 💬 沟通管理 | 飞书、钉钉 | 开会、记录、协调 |
| ⚙️ 执行任务 | Cursor、Claude Code | 编程、设计、执行 |

**问题：** 飞书里开完会、做了记录，真正执行时却在另一个工具里——发生了什么，彼此都不知道。

**解决：** CodeBanana 合二为一——沟通与任务在同一个地方，上下文实时同步，决策即执行。

---

## CodeBanana 三大核心特点

- 🤖 **Agent 为主体** — 系统核心是 Agent，人类与 Agent 协同工作，而非人类主导一切。
- 🏢 **为组织而生** — 支持多人协作，权限管理明确，项目间文件系统隔离，保障安全与效率。
- ⚡ **任务与沟通合一** — 集成群聊、Agent 工作区和文件系统，沟通与任务处理无缝衔接。

---

## 以「项目」为核心

每一个项目，同时是三位一体的存在：

- 🤖 **一个 Agent** — 每个项目自带专属 Agent，具备独立的执行能力，可以完成任务、写代码、处理文件。
- 💬 **一个群聊** — 同时也是一个实时群聊空间，团队成员与 Agent 在同一个上下文中沟通协作。
- 🗂️ **一个 Workspace** — 独立的文件系统与虚拟机，数据安全隔离，Agent 直接在其中读写、运行、部署。

## 系统架构

| 组件 | 说明 |
|------|------|
| 🟣 **Team Agent** | 共享 Agent，同一时间仅一人可操作，聚焦协作。 |
| 🔵 **Private Agent** | 个人专属 Agent，私密空间独立运转。 |
| 🟠 **Discussion** | 人类沟通区，团队讨论与决策实时同步。 |
| 🟢 **Workspace** | 文件与虚拟机，三位一体协作环境的核心底座。 |

---

## 为协作而生

- 👥 **多人实时聊天** — 团队成员在同一个项目空间里自由沟通，Agent 始终在场，随时可被呼叫。
- ⚡ **聊天直接变任务** — 人与人之间的对话，可以一键转发给 Agent 执行，沟通即指令。
- 📤 **文件与消息易于流转** — 系统内的文件和消息可以快速转发、引用和分享，信息不再孤立。
- 🛍️ **Skill 发布与共享** — 自定义能力可发布到 Skill 市场，跨项目、跨团队复用，让组织智慧沉淀下来。

---

## 人与 Agent 消息分离

Agent 工作消息多且密，与人类聊天混在一起，没有人会再去看。

| Tab | 内容 |
|-----|------|
| 💬 **Discussion** | 人与人之间的沟通 — 讨论、决策、协作 |
| 🤖 **Agent** | Agent 执行过程 — 任务、代码、结果 |

两个独立 Tab 分别推送，人类消息不被淹没。

---

## 让 Agent 主动干活（Cron Job）

不只是你问它才动——通过 Cron Job，Agent 按计划自动执行任务：

- ⏰ **定时触发** — 设定时间或周期，Agent 自动醒来执行：每日报告、定期检查、定时提醒。
- 🔁 **事件驱动** — 文件变化、消息到达、条件触发——Agent 监听变化，第一时间响应。
- 🤝 **从被动到主动** — Agent 不再只是工具，而是真正的协作伙伴，主动推进组织目标。

## Skill Market — 让好方法流遍全公司

- 📋 **工种的工作流模板** — Skill 是每个岗位的方法论沉淀，安装即生效。
- ⬆️ **一人做好，上传市场** — 某个人摸索出了高效的工作方式，打包成 Skill 发布，让全公司都能用上。
- 🚀 **好方法病毒式扩散** — 其他人一键安装即可复用，组织最佳实践真正流转起来。

## 企业版管理能力

- 📊 **Dashboard 全局洞察** — 清晰掌握企业大模型使用状况：哪个部门用得好、哪个人是超级个体。
- 🏗️ **组织架构配置** — 按照公司实际结构配置团队与项目，管理层级清晰。
- 🔐 **权限精细管控** — 对每个人的使用权限进行配置与限制，数据安全、合规可控。

## CodeBanana 与 Cursor / Claude Code 的区别

| 维度 | Cursor / Claude Code | CodeBanana |
|------|----------------------|------------|
| 定位 | 个人开发工具 | 团队协作平台 |
| 沟通 | 无，需要切换到飞书/钉钉 | 内置群聊，沟通与任务一体 |
| 协作 | 不支持多人共享 Agent | Team Agent 支持团队共享 |
| 上下文 | 本地/单人 | 云端，团队共享上下文 |
| 组织管理 | 无 | 支持组织架构、权限管控 |
| Skill 复用 | 无 | Skill Market，跨项目复用 |

> CodeBanana 的目标是让每一个组织都能进化成超级组织——用 AI 倍增每一个人的产能。
